## 2. 概要
### 2.1. 画面目的
既存カテゴリーの情報を編集するための画面。カテゴリー名、説明、カラーコード、表示順を編集し、バリデーションを実施した上でデータベースを更新する。

### 2.2. 前提条件
- タスク管理システムにアクセス可能であること
- 編集対象のカテゴリーがデータベースに存在すること

## 6. イベント仕様
### 6.1. 初期表示処理 (`onLoad`)
1. コントローラー `GET /categories/{id}/edit` が呼び出される
2. CategoryService.findById(id) を呼び出し、指定されたカテゴリー情報を取得する
3. カテゴリーが存在しない場合:
   - 404エラー画面（SCR-CMN-001）へ遷移する
4. カテゴリーが存在する場合:
   - 以下のデータをモデルに設定する:
     - `categoryForm`: 既存カテゴリー情報をマッピングした CategoryForm
     - `categoryId`: 編集対象のカテゴリーID
   - ビュー `category/edit.html` をレンダリングする

### 6.2. 入力項目変更時 (`onChange`)
- （特別な処理なし）

### 6.3. ボタン押下時 (`onClick`)
#### キャンセルボタン（B-301）
1. `/categories` へ遷移する
2. 入力内容は破棄される

#### 更新ボタン（B-302）
1. フォーム内の入力値を取得する
2. POST メソッドで `/categories/{id}` へリクエストを送信する
3. サーバーサイドバリデーションを実行する（詳細は「7. 画面チェック仕様」を参照）
4. バリデーションエラーがある場合:
   - BindingResult にエラー情報が格納される
   - ビュー `category/edit.html` を再表示する
   - エラーメッセージが各入力項目の下に表示される（`invalid-feedback`）
   - 入力済みの値は保持される
5. バリデーション成功の場合:
   - CategoryService.update(id, categoryForm) を呼び出してカテゴリーを更新する
   - フラッシュ属性に成功メッセージ「カテゴリーを更新しました」を設定する
   - カテゴリー一覧画面 `/categories` へリダイレクトする
6. カテゴリーが存在しない場合（他ユーザーによる削除等）:
   - 404エラー画面（SCR-CMN-001）へ遷移する
